# NetChecker 🚀

**NetChecker** — это легковесная и быстрая консольная утилита на .NET для комплексной проверки доступности сетевых ресурсов, анализа блокировок (DPI, фильтрация пакетов, TCP RST) и диагностики базовых сетевых протоколов.

Программа разработана для системных администраторов, разработчиков и продвинутых пользователей, которым необходимо в реальном времени отслеживать состояние доступности списков доменов и IP-адресов.

---

## 🛠 Ключевые возможности

* **Мультипротокольный тест:** Проверка базовой связности через TCP (порт 443), UDP (DNS-запрос к 8.8.8.8) и современный транспорт QUIC (HTTP/3).
* **Динамический тест доменов:** Потоковый обход списков ресурсов (`list-*.txt`) с параллельным выполнением двух тестов: ICMP-пинг и веб-доступ по HTTPS с выводом задержки в миллисекундах.
* **Умное определение характера блокировок:** Утилита распознает сбросы соединений (TCP RST во время TLS-handshake, характерные для систем DPI) и фильтрацию пакетов (DROP / Тайм-аут).
* **Масштабируемый тест IP и подсетей:** Парсинг одиночных адресов и CIDR-подсетей (IPv4/IPv6) с многопоточным сканированием через семафоры.
* **Строгий итоговый отчет:** Ресурс считается проблемным и выводится в финальном отчете, если **хотя бы один** из протоколов/тестов завершился неудачно (частичная доступность).
* **Автономное обновление:** Интеграция с GitHub API для проверки новых версий программы и автоматической синхронизации официальных списков.

## ⚡ Быстрый старт и установка
Чтобы начать работу с последней версией программы, выполните следующие шаги:

Скачивание: Перейдите в [репозиторий релизов и скачайте актуальный файл](https://github.com/staryi441/NetChecker/releases/download/1.1.0/NetChecker.exe) NetChecker.exe

Размещение: Создайте любую папку на диске C:\ (например, C:\NetChecker) и поместите туда скачанный файл.

Первичное обновление: Запустите NetChecker.exe В открывшемся меню нажмите кнопку 4 (Проверка обновлений). Программа автоматически определит наличие новой версии, скачает её и обновит исполняемый файл до последней стабильной сборки.

Синхронизация списков: После автоматического перезапуска программы снова выберите пункт 4 в меню. Это действие скачает актуальные конфигурационные списки из репозитория, чтобы утилита была готова к работе с самыми свежими данными блокировок.

Теперь программа полностью настроена и готова к диагностике!

## 📂 Структура рабочей области

Важно: Строки, начинающиеся с символа #, считаются комментариями и полностью игнорируются парсером. Файлы с суффиксом -user.txt защищены от перезаписи при обновлении через GitHub API.

После первого запуска программа создает следующую иерархию папок в своей директории:

```text
├── NetChecker.exe          # Исполняемый файл утилиты
└── list/                   # Директория списков (формат .txt)
    ├── list-general.txt    # Базовые домены глобальных сервисов
    ├── list-google.txt     # Инфраструктура и сервисы Google
    ├── list-discord.txt    # Домены и CDN-сервера Discord
    ├── list-telegram.txt   # Точки доступа и хосты Telegram
    ├── list-user.txt       # Ваш кастомный список доменов для проверки
    ├── ipset-general.txt   # Базовые пулы IP-адресов
    ├── ipset-discord.txt   # Списки IP/подсетей Discord
    ├── ipset-google.txt    # Списки IP/подсетей Google
    ├── ipset-telegram.txt  # Списки IP/подсетей Telegram
    └── ipset-user.txt      # Ваш кастомный список IP и CIDR-подсетей
